### A.2 Ревизия трёх правок препроцессора по эталону

По эталону из этой сессии уже видно:
- «Страница N» в Word — **ПО ЦЕНТРУ**. Наша замена ptab→right-tab двигает её
  вправо = мы сами внесли баг. Эту замену **убрать или переделать** так,
  чтобы результат совпадал с эталоном.
- Цвет/толщина линии: сверить с эталоном (там линия тёмная двойная, не
  светло-серая — наша замена themeShade→808080 может тоже быть неверной).
  Оставить только если рендер БЕЗ неё дальше от эталона, чем с ней.
- Шрифт футера: сверить визуально. Если Liberation Serif ≠ эталон —
  правильный путь: fontconfig-алиас Cambria→Caladea в Docker-образе
  (`/etc/fonts/local.conf` + `fc-cache -f`) вместо подмены имени шрифта в XML.

Итог A.2 — возможно, препроцессор сожмётся до одной правки или исчезнет.
Это нормальный исход: меньше самодеятельности = ближе к Word.

## Часть B — «Запомнить шаблон» реально применяется при повторной загрузке

### Симптом
После «Шаблон запомнен» повторная загрузка ТОГО ЖЕ файла даёт
`matched_template: {traffic_light: "yellow", candidates_count: 0}` —
матчер видит **ноль** шаблонов в хранилище. Проблема на этапе
«remember сохранил ≠ analyze нашёл», а не в качестве матчинга.

### B.1 Диагностика (по порядку, с выводом в отчёт)

1. После `POST /v1/me/templates/remember` — залистить tenant-scoped storage
   (тот же `TenantScopedStorage`, что использует analyze): реально ли лежит
   `me/{tenant_id}/templates/{id}.json`? Сверить префиксы записи и чтения
   ПОСИМВОЛЬНО (типовой баг: `me/` vs `me//`, uid vs tenant_id из разных полей).
2. Если файл лежит — проверить `list_templates(tenant_storage)` из analyze-пути:
   возвращает ли он этот шаблон? Если нет — искать где analyze создаёт storage
   с другим префиксом/бакетом (test vs prod config).
3. Если list возвращает — прогнать `find_matching_templates` на повторном
   fingerprint: совпадают ли `page_count`, `header_simhash`, `chars_per_page`
   между сохранённым и свежим? DOCX-конвертация детерминирована, fingerprint
   обязан совпасть 1:1 → score должен быть green.
4. В `/v1/me/analyze` добавить в `pipeline_debug` поля:
   `templates_storage_prefix`, `templates_listed_count`,
   `best_match_score_details` — чтобы такие баги дальше диагностировались
   из DevTools без раскопок.

### B.2 Фикс
По результату B.1 — точечный. Не переписывать matcher, не менять формат
шаблона, не добавлять БД: хранилище GCS + TenantScopedStorage — достаточная
«база», проблема в стыковке.

### B.3 Критерий готовности части B
На test: загрузка документа → «Запомнить шаблон» → повторная загрузка того же
файла → в ответе analyze `from_template: true`, `traffic_light: "green"`,
LLM-шаги пропущены (в `timings_ms` нет pipeline LLM-времени), подписи встали
в те же места. Проверить и обратное: «Забыть» → повторная загрузка снова идёт
через LLM.
